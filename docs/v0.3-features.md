# javahome 0.3 feature notes

Version 0.3 focuses on user experience and platform coverage.

## New commands

```text
javahome select
javahome activate
javahome completion bash
javahome completion zsh
javahome completion fish
javahome completion powershell
```

## Interactive selection

`javahome select` shows discovered JDKs as a numbered list. The user enters the number of the desired JDK. It supports the same activation modes as `use`:

```bash
javahome select
javahome select --global --shell bash
javahome select --project
```

## Project activation

Create a project config:

```bash
javahome use 17 --project
```

Activate it later:

```bash
eval "$(javahome activate --shell bash)"
```

Fish:

```fish
javahome activate --shell fish | source
```

PowerShell:

```powershell
javahome activate --shell powershell | Invoke-Expression
```

## Shell completions

Generate shell completions:

```bash
javahome completion bash
javahome completion zsh
javahome completion fish
javahome completion powershell
```

## Native discovery

`javahome` now adds native discovery hints where available:

- macOS: `/usr/libexec/java_home -V`
- Linux: `update-alternatives --query java` and `update-java-alternatives --list`
- Windows: common JDK vendor Registry keys

## Doctor conflict hints

`javahome doctor` now checks for common Java version managers:

- SDKMAN
- jEnv
- asdf
- mise

If one appears active in `PATH`, doctor reports a warning because that tool may override `JAVA_HOME` or `PATH` after `javahome`.

## Release workflow

Pushing a tag like `v0.3.0` triggers `.github/workflows/release.yml`, which builds Linux, macOS, and Windows binaries and attaches them to a GitHub Release.
